Добро пожаловать на первый шаг проектирования нейросетей. В глубоком обучении все данные — от картинок до весов моделей — представляются в виде многомерных массивов (тензоров). Этот урок закладывает базу для низкоуровневой работы с ними на примере библиотеки PyTorch.

🔑 Ключевое правило ML-инженера: Любое изменение формы тензора не должно перемешивать или искажать исходную физическую структуру данных. Код должен быть вычислительно эффективным (без лишнего копирования в памяти).

1. Ранги тензоров: от скаляра до N-мерного массива

  • Скаляр (Ранг 0): Простое число. Форма (shape): []
  • Вектор (Ранг 1): Одномерный массив. Форма: [размер]
  • Матрица (Ранг 2): Двумерная таблица. Форма: [строки, столбцы]
  • Тензор (Ранг 3+): Многомерный массив. Пример: [батч, каналы, высота, ширина]
import torch

scalar = torch.tensor(3.1415)
print(f"Скаляр: {scalar}, Ранг: {scalar.ndim}, Форма: {scalar.shape}")

vector = torch.tensor([1.0, 2.0, 3.0])
print(f"Вектор: Форма {vector.shape}")

matrix = torch.tensor([[1, 2], [3, 4]])
print(f"Матрица: Форма {matrix.shape}")

images = torch.randn(32, 3, 224, 224)
print(f"4D Тензор: Форма {images.shape}")

2. Базовые операции и тензорное произведение

▸ Поэлементные операции

a = torch.tensor([1, 2])
b = torch.tensor([3, 4])
print("Сумма:", a + b)       # [4, 6]
print("Умножение:", a * b)   # [3, 8]

▸ Матричное умножение

matrix_A = torch.randn(2, 3)  # 2×3
matrix_B = torch.randn(3, 4)  # 3×4
result = matrix_A @ matrix_B
print("Форма:", result.shape)  # 2×4

Тензорное произведение (einsum)

batch_A = torch.randn(10, 2, 3)
batch_B = torch.randn(10, 3, 4)
result = torch.einsum('bik,bkj->bij', batch_A, batch_B)
print("Форма после einsum:", result.shape)

3. Изменение размерностей без потери структуры

️ Опасность: Никогда не используйте reshape для транспонирования осей! Он берёт элементы подряд из памяти и ломает пространственные связи данных.

▸ View vs Reshape

x = torch.arange(12)  # [0, 1, 2, ..., 11]
matrix = x.view(3, 4)  # Без копирования
matrix2 = x.reshape(4, 3)  # Безопасно, но может копировать

▸ Permute: перестановка осей

images = torch.randn(32, 3, 64, 64)  # [B, C, H, W]
images_vis = images.permute(0, 2, 3, 1)  # [B, H, W, C]
print(images_vis.shape)

▸ Squeeze / Unsqueeze

img = torch.randn(3, 224, 224)  # [C, H, W]
batch = img.unsqueeze(0)  # [1, C, H, W]
restored = batch.squeeze()  # [C, H, W]

import torch

def prepare_for_attention(embeddings, num_heads=4):
    B, L, D = embeddings.shape
    # Шаг 1: Разбиваем Hidden_Dim на головы
    # [B, L, num_heads, Head_Dim]
    x = embeddings.view(B, L, num_heads, D // num_heads)
    
    # Шаг 2: Меняем оси, чтобы головы стали вторым измерением
    # [B, num_heads, L, Head_Dim]
    x = x.permute(0, 2, 1, 3)
    return x

if __name__ == "__main__":
    x = torch.randn(2, 10, 64)
    print("Исходная форма:", x.shape)
    
    result = prepare_for_attention(x, num_heads=4)
    print("Целевая форма:", result.shape)
    assert result.shape == (2, 4, 10, 16), "Ошибка формы!"

📚 Multi-Head Attention: Теория

Что такое Multi-Head Attention?

Механизм внимания (Attention) — это способ для модели «фокусироваться» на разных частях входной последовательности при обработке каждого элемента.

🔑 Основная идея:

Вместо того чтобы обрабатывать всё скрытое представление (Hidden_Dim) целиком, мы:

  1. Разбиваем его на num_heads частей (голов)
  2. Каждая голова учится обращать внимание на разные аспекты данных
  3. Результаты всех голов объединяются

📐 Математика преобразования:

Вход: [Batch, Seq_Len, Hidden_Dim] = [2, 10, 64]

Шаг 1 (view): Разбиваем 64 на 4 головы по 16
  → [2, 10, 4, 16]

Шаг 2 (permute): Меняем оси (1↔2)
  → [2, 4, 10, 16]

Итог: [Batch, num_heads, Seq_Len, Head_Dim]

🎯 Зачем это нужно?

  • Параллелизация: 4 головы работают независимо
  • Разные паттерны: Одна голова может учиться синтаксису, другая — семантике, третья — позиционным связям
  • Эффективность: Меньше вычислений, чем если бы обрабатывали всё вместе
