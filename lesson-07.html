Многослойная линейная сеть эквивалентна одному полносвязному слою. Чтобы сеть могла приближать сложные нелинейные зависимости, между слоями вводят функции активации. А чтобы глубокие сети обучались стабильно и быстро, распределение промежуточных значений нужно контролировать с помощью слоев нормализации.

Проблема внутреннего сдвига ковариат (Internal Covariate Shift): По мере обновления весов на этапе обучения распределение активаций на входе каждого глубокого слоя постоянно меняется. Это заставляет слои непрерывно подстраиваться под новые условия, замедляя сходимость. Нормализация удерживает среднее и дисперсию сигналов в стабильных рамках.

1. Современные функции активации: GeLU и Swish

На смену классическому ReLU пришли функции с гладким изгибом около нуля, которые не зануляют градиенты полностью для отрицательных значений, что предотвращает проблему "умирающих нейронов" (Dying ReLU):

  • GeLU (Gaussian Error Linear Unit): Активация на основе функции распределения Гаусса. Стандарт для современных трансформеров (BERT, GPT).
    GeLU(x) ≈ 0.5 · x · (1 + tanh(√(2/π) · (x + 0.044715 · x³)))
  • Swish / SiLU: Разработана Google с помощью автоматического поиска архитектур.
    Swish(x) = x · σ(βx)
import torch
import torch.nn as nn

# Реализация GeLU аппроксимацией на чистом PyTorch
def custom_gelu(x):
    return 0.5 * x * (1.0 + torch.tanh(torch.sqrt(torch.tensor(2.0 / torch.pi)) * (x + 0.044715 * torch.pow(x, 3))))

test_tensor = torch.tensor([-2.0, -0.5, 0.0, 2.0])
print("Кастомный GeLU:", custom_gelu(test_tensor))

2. Сравнение подходов к нормализации

Выбор оси, по которой считаются среднее значение ($\mu$) и дисперсия ($\sigma^2$), коренным образом меняет поведение модели:

  • Batch Normalization (BatchNorm): Считает статистики по всему батчу для каждого канала отдельно. Идеален для сверточных сетей. Сильно зависит от размера батча (плохо работает при Batch Size < 4). Хранит бегущие средние (running stats) для инференса.
  • Layer Normalization (LayerNorm): Считает статистики по всем фичам одного конкретного примера. Незаменим в NLP и Трансформерах. Не зависит от размера батча и ведет себя одинаково в train/eval.
  • Group Normalization (GroupNorm): Компромисс между двумя предыдущими методами. Делит каналы одного примера на группы и нормализует внутри них. Используется в задачах компьютерного зрения с тяжелыми моделями (например, в Диффузии/U-Net).

3. Написание кастомного слоя Layer Normalization

Любой слой нормализации содержит два обучаемых параметра: масштаб (gamma) и сдвиг (beta). Они позволяют сети вернуть исходное распределение, если это потребуется для минимизации лосса.

class CustomLayerNorm(nn.Module):
    def __init__(self, normalized_shape, eps=1e-5):
        super().__init__()
        self.eps = eps
        # Инициализируем обучаемые параметры
        self.gamma = nn.Parameter(torch.ones(normalized_shape))
        self.beta = nn.Parameter(torch.zeros(normalized_shape))

    def forward(self, x):
        # x имеет форму [Batch_Size, Sequence_Length, Hidden_Dim]
        
        # Вычисляем среднее и дисперсию по последней размерности (фичам)
        mean = x.mean(dim=-1, keepdim=True)
        var = x.var(dim=-1, keepdim=True, unbiased=False)
        
        # Центрируем и масштабируем данные до среднего=0 и дисперсии=1
        x_norm = (x - mean) / torch.sqrt(var + self.eps)
        
        # Применяем аффинное преобразование с обучаемыми весами
        return self.gamma * x_norm + self.beta

✅ Готовое решение (solution_rms.py) 
import torch
import torch.nn as nn

class CustomRMSNorm(nn.Module):
    def __init__(self, normalized_shape, eps=1e-5):
        super().__init__()
        self.eps = eps
        self.gamma = nn.Parameter(torch.ones(normalized_shape))

    def forward(self, x):
        # Считаем среднеквадратичное (Root Mean Square)
        # x.pow(2).mean(-1, keepdim=True) - это E[x^2]
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        
        # Нормализуем, деля на RMS, и масштабируем гаммой
        return x / rms * self.gamma

if __name__ == "__main__":
    rms = CustomRMSNorm(64)
    x = torch.randn(2, 10, 64)
    out = rms(x)
    print("Output shape:", out.shape)
📚 Почему RMSNorm быстрее и лучше?

Экономия вычислений

В LayerNorm мы делаем два шага:

  1. Вычитаем среднее: $(x - \mu)$
  2. Делим на стандартное отклонение: $/ \sigma$

В RMSNorm мы пропускаем первый шаг (вычитание среднего). Исследования (Zhang & Sennrich, 2019) показали, что перенос среднего значения не так важен для качества обучения больших моделей, как масштабирование дисперсии.

💡 Главный плюс: Отсутствие операции вычитания среднего делает слой на 7-64% быстрее (в зависимости от железа) и потребляет меньше видеопамяти. Именно поэтому Llama 3 и Mistral используют RMSNorm вместо LayerNorm.
